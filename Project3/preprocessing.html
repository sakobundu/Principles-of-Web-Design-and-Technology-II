Preprocessing.

Different ways to implement preprocessing (Client-side vs Server-side):

Client-side prepossessing can be implemented using JavaScript, and HTML scripts. Most client interaction with a website is input and output, that where JavaScript comes in. It runs the script on the client’s system when the website is loaded. Server-side preprocessing can be implemented using backend languages such as PHP. The script is running on the server which hosts the website and sends information to the front-end through HTML code.

Mechanisms used in LESS:

Mixins:

Mixins are a group of CSS properties that allow you to use properties of one class for another class and includes class name as its properties. It can store multiple values and can be reused in the code whenever necessary.

Variables:

LESS allows variables to be defined with an @ symbol. The Variable assignment is done with a colon(:).

Functions:

Less provides a variety of functions which transform colors, manipulate strings and do math. Function has different types: Logical, String, List, Math, Type, Misc, Color Definition, Color Channel, Color Operation, and Color Blending.

Nested variables:

Nested variables are the combination of different variables in a condensed form, so it does not take up a lot of space in the IDE.

Operators:

Operators in LESS can perform varieties of arithmetical operations using any number, color or variable.
